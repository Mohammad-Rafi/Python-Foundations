⚡ Flash Notes – Concepts by Category ~ Input/Output Operations

- Multi-line Input
  - Use a loop until an empty line is entered.
  - Stores multiple lines in a list.

- sys.stdin
  - Used for reading input directly from standard input stream.
  - Useful for bulk/multi-line input.


📤 Output Operations
- print() function
  - Displays output to console.
  - Accepts multiple arguments.
  - Parameters:
    - sep → custom separator between values.
    - end → custom ending string.
    - file → redirect output to a file.

- Formatting Output
  - f-strings → modern, clean formatting.
    Example: print(f"Pi = {3.14159:.2f}")

  - format() method → older but flexible.
    Example: "Name: {}, Age: {}".format(name, age)

  - Alignment and width specifiers for tables.
    🔹 Alignment Options
        - <  → Left align
        - >  → Right align
        - ^  → Center align

    🔹 Width Specifiers
        - Define minimum width of the field.
        - Example: :<10 → left align in a space of 10 characters.
        - If content is shorter, spaces are added automatically.


📂 File Input/Output
- Opening Files
  - open(filename, mode)
  - Modes:
    - "r" → read
    - "w" → write (overwrite)
    - "a" → append
    - "b" → binary
    - "r+" → read & write

- Reading Files
  - file.read() → entire file
  - file.readline() → one line
  - file.readlines() → list of lines

- Writing Files
  - file.write("text") → writes string to file

- Closing Files
  - file.close() → manually close file
  - with open(...) as f: → auto-closes file

- Redirecting Output
  - print("Hello", file=f) → writes directly to file


🧮 Expression Evaluation
- eval() function
  - Evaluates a string as a Python expression.
  - Use with caution → security risks.
  - Safe usage: restrict builtins.
  - Example:
    expr = input("Enter math expression: ")
    result = eval(expr, {"__builtins__": None}, {})


🔑 Key Points Recap
- Input is always string → convert if needed.
- print() can be customized with sep, end, file.
- Use f-strings for modern formatting.
- File I/O requires open/close; prefer with for safety.
- eval() should be sandboxed for security.
